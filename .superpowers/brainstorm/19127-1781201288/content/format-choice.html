What format should the visualizations take?

This determines where users will see them and how much visual richness is possible.

A

Mermaid diagrams in README.md

GitHub renders Mermaid natively — flowcharts, sequence diagrams, component trees appear as clean vector graphics right in the README. Zero extra files, always in sync with code.

```mermaid
flowchart LR
  Input --> Canvas --> PNG
```
B

Standalone HTML visualization page

A beautiful docs/overview.html with animated SVG diagrams, interactive elements, and rich design. Can be opened in any browser or hosted as a GitHub Pages doc site.

✦ Animated rendering pipeline diagram
✦ Interactive component explorer
✦ Visual paper-to-output flow
C

Both — Mermaid in README + rich HTML page

README gets clean inline diagrams for quick orientation. A separate visual page goes deeper with animated flow, architecture breakdown, and a "how it works" walkthrough. Recommended — covers both quick-glance and deep-dive readers.
